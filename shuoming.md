# **学习系统 MVP 架构设计（H5 小程序版）**

## **一、整体架构**

1. **用户端：H5 小程序 / Web 页面**

   * 浏览学习资料
   * 做题与考试
   * 查看学习进度
2. **管理端：Web 管理后台**

   * 上传资料
   * 管理题库与试卷
   * 查看学习记录
3. **服务端：Go + Gorm + MySQL**

   * 提供 API 服务
   * 用户认证（JWT / session）
   * 课程、题库、考试、进度记录管理
4. **数据层：MySQL + 文件存储**

   * 学习资料、视频、文件
   * 用户学习记录与考试记录

---

## **二、核心功能模块**

| 模块          | 功能（MVP）                      |
| ----------- | ---------------------------- |
| **内容学习**    | 列表 / 详情 / 完成状态               |
| **考试系统**    | 题库管理 / 试卷管理 / 用户做题 / 自动评分    |
| **专题/培训计划** | 可选，支持组合资料 + 考试               |
| **学习记录**    | 用户完成状态 / 考试成绩                |
| **管理后台**    | 资料 CRUD / 题库 CRUD / 用户学习数据查看 |

---

## **三、角色（精简版）**

* **用户（User）**：H5 小程序端访问
* **管理员（Admin）**：Web 管理后台
* **系统管理员（可选）**：管理管理员账号或系统配置

---

## **四、数据结构（MVP）**

| 表                   | 功能                     |
| ------------------- | ---------------------- |
| users               | 用户信息（ID / 昵称 / openid） |
| admins              | 管理员账号                  |
| materials           | 学习资料（图文 / 视频 / 文件）     |
| questions           | 题库                     |
| exams               | 试卷                     |
| exam_records        | 用户考试记录                 |
| course_records      | 学习记录                   |
| topics（可选）          | 学习专题                   |
| topic_materials（可选） | 专题对应资料                 |
| topic_exams（可选）     | 专题对应考试                 |

---

---

## **六、MVP 特点**

* 可快速上线，功能闭环：
  **资料上传 → 用户学习 → 做题 → 自动评分 → 管理员查看**
* 可跨平台（H5 / 微信 / 浏览器）
* 后续扩展简单：

  * 添加更多角色（系统管理员、督导等）
  * 增加统计报表、消息推送、学习计划、积分/成长体系
